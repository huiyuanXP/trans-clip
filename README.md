# trans-clip

把手机上的输入（含语音输入法）实时同步到电脑端，方便你在电脑里“只管粘贴”。

## 当前进度（已可用）

- ✅ 手机输入端：输入/语音输入后实时推送到服务端
- ✅ 电脑接收端：实时显示“最新内容”，并提供“一键复制到剪贴板”
- ✅ 局域网使用：接收端会自动检测电脑的局域网 IPv4（macOS/Windows），自动生成“手机打开链接”
- ⚠️ 自动写入剪贴板：受浏览器策略限制，通常需要先点一次“复制到剪贴板”激活；更稳定的方式是用电脑端小代理（见下）

## 能不能实现？

能实现“手机输入 → 电脑接收 → 复制到剪贴板”这条链路；但要注意：

- **纯网页**在多数浏览器里**无法后台持续、无感**地自动改写系统剪贴板（通常需要用户手势/页面保持激活）。
- 因此本项目提供两种接收方式：
  - **网页接收端**：尽力自动写入剪贴板；失败时提供“一键复制”兜底。
  - **电脑小代理（Node 脚本）**：可靠后台写入剪贴板（不受浏览器限制），更接近你想要的“打开就不管它”。

## 快速开始（同一局域网）

### 1) 启动服务端（在电脑）

```bash
npm i
npm run dev
```

默认监听 `0.0.0.0:8787`（同网段手机可访问）。如果端口冲突或你想换端口，可用环境变量修改：

- `PORT=8787`
- `HOST=0.0.0.0`

### 2) 电脑打开接收端

在电脑浏览器打开：

- `http://127.0.0.1:<PORT>/receiver.html`

点“生成新房间”。因为这里是 `127.0.0.1/localhost`（只代表“这台电脑自己”），所以需要在页面里填写一次“给手机用的访问地址”，例如：

- `http://192.168.1.23:<PORT>`

现在接收端会尝试**自动检测**电脑的局域网 IP 并填好；你也可以手动改。然后点“一键复制”把“手机打开链接”复制到手机浏览器即可。

### 3) 手机上打开输入端

- 在手机浏览器打开接收端页面里给的链接（会自动带上房间号）
- 或打开：`http://<电脑IP>:<PORT>/sender.html` 然后输入房间号

在输入框里打字或用语音输入，文本会实时推送到电脑端。

## 可选：更稳定的剪贴板写入（推荐）

网页方案受浏览器限制。如果你想要更接近“后台自动写入剪贴板”，在电脑运行：

```bash
npm run agent -- --server http://127.0.0.1:8787 --room 123456
```

- `--room` 用接收端页面显示的 6 位房间号
- 运行后它会把收到的文本直接写入系统剪贴板，你在任何地方直接粘贴即可

## 说明与安全

- 房间与文本都只保存在服务端内存里（重启即消失），不做持久化。
- 如果你把服务端暴露到更大的网络，6 位房间号有被猜中的风险；建议仅在局域网使用，或自行加一层鉴权/反向代理。
